## 1. Rủi ro
- Rủi ro về bảo mật:
  + Sandbox Escaping: User có thể muốn hack, phá hệ thống
  + Tấn công mạng nội bộ: User dùng mã độc để tấn công các dịch vụ nội bộ
  + Tiêu thụ tài nguyên trái phép: User có thể nộp code để đào coin...
- Rủi ro về Hiệu suất và mở rộng:
  + Tắc nghẽn hàng đợi: Nhiều người nộp bài cùng lúc
  + Đo lường thời gian không nhất quán: CPU quá tải và user có thể bị đánh lỗi TLE một cách oan uổng
- Rủi ro về chi phí hạ tầng: Nếu không tối ưu hoá quá trình khởi tạo Sandbox, chi phí thuê Cloud sẽ bị đội lên rất cao
## 2. Edge cases
- Answer thiếu thừa 1 dấu cách hoặc dấu xuống dòng nhưng vẫn chấm là sai
- Sai số rất nhỏ
- Máy chấm tự làm khó thì code của thí sinh viết loằng ngoằng, mất nhiều thời gian biên dịch, từ đó làm hệ thống bị đơ cứng
